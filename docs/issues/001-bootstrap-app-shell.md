## What to build

Build the initial single-page bookmark manager shell so the app has a usable structure before feature logic is layered in. This slice should deliver the page layout, semantic sections, responsive styling, and the initial empty-state presentation for a user who has no bookmarks yet.

This issue should establish the visible product shape described in the Solution and Implementation Decisions sections of the PRD without yet implementing full CRUD behavior.

## Acceptance criteria

- [ ] The page renders a clear title, short description, bookmark form region, search region, and bookmark list region.
- [ ] The layout is usable on both desktop and mobile screen sizes.
- [ ] When there are no bookmarks, the app shows a clear empty state instead of an empty container.
- [ ] The markup uses semantic HTML suitable for future form and list interactions.
